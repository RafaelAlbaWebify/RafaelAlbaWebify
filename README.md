## How I Approach Support Work

1. Define the symptom clearly.
2. Identify scope: one user, many users, one device, one network, one service or a wider incident.
3. Collect evidence before changing configuration.
4. Separate identity, endpoint, network, policy and application layers.
5. Document the investigation path, not only the final fix.
6. Turn repeat issues into checklists, runbooks or small tools.
