## Creation Rule For New Flagship Repos

Do not create an empty flagship repo just for branding.

A new flagship repo should exist only when there is at least:

```text
working MVP or useful scaffold
  -> README with first-person positioning
  -> clear safety/boundaries
  -> one sample scenario
  -> one roadmap
  -> no private data
```
